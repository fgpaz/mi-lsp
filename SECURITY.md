# Security Policy

## Supported versions

`mi-lsp` currently supports security fixes for the latest public release only.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting a vulnerability

Do not open public GitHub issues for security reports.

Please use GitHub's private vulnerability reporting flow for this repository:

- Open the repository on GitHub
- Go to the Security tab
- Choose "Report a vulnerability"

Reports sent through GitHub private vulnerability reporting are the official
channel for this project.

## What to include

Please include as much of the following as you can:

- A short description of the issue
- Affected command or feature
- Impact and realistic attack scenario
- Reproduction steps or proof of concept
- Environment details such as OS, architecture, and `mi-lsp` version
- Any suggested remediation if you have one

## Response targets

- Initial acknowledgement target: within 48 hours
- Follow-up and remediation timeline: best effort, depending on severity and maintainer availability

## Coordination

- Please avoid public disclosure until a fix or mitigation is available.
- If the report is accepted, the maintainer will coordinate remediation and release timing through the private advisory workflow.
