## Reglas transversales

- Toda respuesta debe incluir `backend`, `warnings`, `stats` y `truncated`.
- El daemon nunca es requisito para que la CLI funcione.
- El worker .NET nunca se descarga en forma implicita durante una consulta.
- El indice repo-local almacena catalogo liviano de codigo y un grafo documental reutilizable para `nav ask`.
- `nav ask` rankea docs primero y usa codigo como evidencia, no al reves.
- `nav ask` y `nav pack` deben bloquear si la gobernanza del repo no es valida.
- `nav service` devuelve evidencia estructurada y no un score fuerte de completitud.
- `client_name` y `session_id` son metadatos operativos de trazabilidad, no parte del payload funcional del usuario.
- Si el request omite `workspace`, la resolucion debe priorizar el root registrado que contiene el `caller_cwd`; solo despues puede caer a `last_workspace`, con warning visible.
- `nav.search` debe auto-curarse ante regex invalido y devolver warning/hint util antes que error duro cuando existe fallback literal seguro.
- `nav.ask` debe servir por camino directo por default; el daemon queda reservado para queries semanticas o compuestas que realmente se benefician del warm state.
- `workspace list` y `nav.workspace-map` deben separar explicitamente modo rapido vs expansion profunda.
